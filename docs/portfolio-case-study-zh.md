# Video Mind AI 项目说明

## 项目简介

Video Mind AI 是一个面向短视频创作者的 AI 视频解析与创作工作台，支持将参考视频转化为可复用的创作素材，包括字幕转写、内容摘要、场景拆解、脚本改写和提示词生成。

它的目标不是只给出一段总结文本，而是把参考视频整理成后续可继续创作、编辑和复用的结构化结果。

## 核心流程

`参考视频导入 -> 转写与摘要 -> 场景拆解 -> AI 人设改写 -> 导出脚本与提示词`

## 主要能力

### 1. 视频理解

- 支持本地视频上传
- 支持公开视频链接导入
- 异步生成 transcript 和 summary
- 展示分析状态与错误信息

### 2. 结构化场景拆解

系统会围绕视频内容生成可复用的场景结果，包括：

- 场景标题
- 动作描述
- 对白片段
- 镜头类型
- 情绪线索
- 可复用的内容 hook

这些结果会落库保存，而不是停留在一次性的文本响应里。

### 3. AI 人设改写

在完成视频理解后，系统会基于已有分析结果生成新的创作草稿，包括：

- 脚本草稿
- 分镜草稿
- Prompt Bundle

这样用户可以围绕自己的 AI 角色继续做后续生成，而不是从零重新整理素材。

## 技术实现

### 异步任务链路

基于 RocketMQ 将字幕转写、摘要生成、场景拆解等长耗时任务从上传请求中解耦，上传接口只负责接收任务，后续分析由消费者异步处理。

这样可以避免把长时间 AI 处理直接压在请求链路上，提升接口响应速度和整体稳定性。

### 并发控制与限流

基于 Redis 和 Redisson 对分析任务做并发控制：

- 使用分布式锁拦截重复提交
- 使用全局限流控制高频分析请求

这样可以减少重复分析带来的资源浪费，也能避免高峰期请求把后端任务链路压垮。

### 结构化存储模型

项目围绕创作工作流设计了多层数据结构：

- `media_files` 保存项目基础信息、来源、摘要和状态
- `scene_segments` 保存场景级拆解结果
- `rewrite_drafts` 保存改写后的脚本、分镜和提示词内容

这样做的价值是让 AI 输出能被前端稳定展示、编辑和复用，而不是只生成一段难以管理的文本。

## 前端工作台设计

前端不是单纯的上传页，而是围绕“项目”组织完整工作流，主要包括：

- 上传入口
- 项目列表
- 项目详情
- 场景拆解结果区
- 改写与导出区

这种结构可以让用户围绕同一个参考视频连续完成理解、拆解和改写，而不是在多个零散页面之间来回切换。

## 技术栈

- 前端：Vue 3、Vite
- 后端：Spring Boot、MyBatis Plus、MySQL
- 中间件：Redis、Redisson、RocketMQ、MinIO
- AI 与媒体处理：FFmpeg、yt-dlp、Aliyun ASR、DeepSeek-compatible API
